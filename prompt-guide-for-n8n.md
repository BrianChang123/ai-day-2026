# AI에게 n8n 워크플로우 만들어달라고 할 때, 프롬프트 작성 가이드

## 왜 프롬프트가 중요한가?

AI에게 "엑셀 자동화해줘"라고 하면, AI도 뭘 해야 할지 모릅니다.
사람한테 일을 시킬 때도 구체적으로 말해야 잘 되는 것처럼,
**AI에게도 구체적으로 설명할수록 좋은 결과가 나옵니다.**

---

## 핵심 원칙: "신입사원에게 인수인계한다고 생각하세요"

프롬프트를 쓸 때 가장 좋은 마음가짐은 이것입니다:

> **"이 업무를 처음 하는 신입사원에게 설명한다"**

신입사원한테 "보고서 만들어"라고만 하면 안 되잖아요?
어떤 데이터를, 어디서 가져와서, 어떤 형식으로, 누구한테 보내는지 알려줘야 합니다.

AI도 똑같습니다.

---

## 프롬프트 5요소 프레임워크

좋은 프롬프트에는 이 5가지가 들어갑니다:

```
① 상황 (Context)    — 어떤 업무인지, 배경이 뭔지
② 입력 (Input)      — 어디서 데이터가 오는지, 어떤 형태인지
③ 처리 (Process)    — 데이터를 어떻게 가공해야 하는지
④ 출력 (Output)     — 결과물이 뭔지, 어떤 형식인지
⑤ 전달 (Delivery)   — 결과를 누구에게, 어떻게 보내는지
```

### 기억하기 쉬운 방법

> **"어디서 → 뭘 가져와서 → 어떻게 처리하고 → 뭘 만들어서 → 어디로 보내"**


---

## 실전 예시 1: 이메일 첨부파일 → 엑셀 정리

### ❌ 나쁜 프롬프트

```
이메일 오면 엑셀에 정리해줘
```

→ AI 반응: "어떤 이메일? 어떤 엑셀? 뭘 정리하라는 건지..."

### ⚠️ 보통 프롬프트

```
Outlook에서 이메일이 오면 첨부된 엑셀 파일을 구글 시트에 정리해줘
```

→ 방향은 맞지만, 어떤 이메일인지, 어떤 시트인지, 어떻게 정리하는지 모름

### ✅ 좋은 프롬프트

```
n8n 워크플로우를 만들어줘.

[상황]
CXM 팀에서 매일 이메일로 받는 주문 데이터를 구글 시트에 정리하는 업무야.
지금은 담당자가 수작업으로 하고 있어서 하루 30분 정도 걸려.

[입력]
- Outlook에서 제목에 "주문현황"이 포함된 이메일이 도착하면 트리거
- 첨부파일은 .xlsx 형식이야
- 첨부파일 안에는 주문번호, 고객명, 상품코드, 수량, 금액 컬럼이 있어

[처리]
- 첨부파일에서 위 5개 컬럼의 데이터를 추출해줘
- 빈 행이 있으면 건너뛰고, 금액이 0인 행도 제외해줘

[출력]
- 구글 시트 "2024_주문취합" 시트의 "raw_data" 탭에 행을 추가해줘
- 추가할 때 A열에 오늘 날짜를 자동으로 넣어줘

[전달]
- 시트에 데이터가 추가되면 Slack #cxm-알림 채널에
  "주문 데이터 {N}건이 업데이트되었습니다" 메시지를 보내줘
```

### 왜 이게 좋은가?

| 요소 | 나쁜 프롬프트 | 좋은 프롬프트 |
|------|-------------|-------------|
| 상황 | 없음 | CXM 팀, 매일 반복, 30분 소요 |
| 입력 | "이메일" | Outlook, 제목 필터, .xlsx, 5개 컬럼 명시 |
| 처리 | 없음 | 빈 행 건너뛰기, 금액 0 제외 |
| 출력 | "엑셀" | 구체적 시트명, 탭명, 날짜 컬럼 |
| 전달 | 없음 | Slack 채널명, 메시지 형식 |

## 실전 예시 2: 미제출자 자동 리마인더

### ❌ 나쁜 프롬프트

```
안 보낸 사람한테 알림 보내줘
```

### ✅ 좋은 프롬프트

```
n8n 워크플로우를 만들어줘.

[상황]
매주 금요일까지 각 팀에서 주간 보고를 구글 시트에 입력해야 해.
매번 안 넣은 사람을 확인하고 직접 독촉하는 데 시간이 걸려.

[입력]
- 구글 시트 "주간보고_2024" 시트
- B열: 팀명, C열: 담당자명, D열: 이메일, E열: 이번주 입력 여부 (Y/N)
- 매주 금요일 오전 10시에 자동으로 실행

[처리]
- E열이 "N"인 행을 필터해서 미입력자 목록을 만들어줘

[출력]
- 미입력자 각각에게 개별 이메일을 보내줘
- 제목: "[리마인더] 주간 보고 미입력 안내"
- 본문: "{담당자명}님, 이번 주 주간 보고가 아직 입력되지 않았습니다.
        금요일 오후 3시까지 입력 부탁드립니다."

[전달]
- 발송 완료 후, Slack #관리자-알림 채널에
  "이번 주 미입력자 {N}명에게 리마인더를 발송했습니다.
   미입력자: {이름 목록}" 형태로 요약 메시지를 보내줘
```

## 실전 예시 3: 매일 자동 보고서 생성

### ❌ 나쁜 프롬프트

```
매일 보고서 자동으로 만들어줘
```

### ✅ 좋은 프롬프트

```
n8n 워크플로우를 만들어줘.

[상황]
매일 아침 전일 매출 데이터를 취합해서 팀장에게 보고하는 업무야.
지금은 매일 아침 30분 걸려서 수작업으로 하고 있어.

[입력]
- 매일 오전 8시 30분에 자동 실행
- 구글 시트 "일일매출" 시트에서 어제 날짜에 해당하는 행만 가져와
- 컬럼: 날짜, 매장명, 상품카테고리, 매출액, 판매수량

[처리]
- 매장별 매출 합계를 계산해줘
- 상품카테고리별 판매수량 합계도 계산해줘
- 전일 대비 증감률도 계산해줘 (2일 전 데이터와 비교)

[출력]
- 위 내용을 아래 형식의 텍스트 보고서로 만들어줘:

  📊 일일 매출 보고서 ({어제 날짜})

  ■ 매장별 매출
  - {매장명}: {매출액}원 (전일 대비 {증감률}%)

  ■ 카테고리별 판매
  - {카테고리}: {수량}건

  ■ 총 매출: {합계}원

[전달]
- 위 보고서를 Slack #daily-report 채널에 보내줘
- 동시에 팀장(이메일: manager@company.com)에게도 이메일로 보내줘
- 이메일 제목: "[일일매출] {어제 날짜} 매출 보고서"
```

## 실전 예시 4: 클레임 접수 자동화 (CXM 팀 실제 사례 기반)

### ❌ 나쁜 프롬프트

```
클레임 접수 자동화해줘
```

### ✅ 좋은 프롬프트

```
n8n 워크플로우를 만들어줘.

[상황]
해외 스토어에서 불량 교환 요청(클레임)이 들어오면
CXM 팀이 SAP에 출고 처리하고, 해외팀에 이메일을 보내는 업무야.
지금은 Slack, 엑셀, SAP, 이메일을 오가며 수작업으로 처리하고 있어.

[입력]
- Slack에서 /claim 슬래시 명령어로 시작
- 입력 양식(Modal)을 띄워서 아래 정보를 받아:
  - 클레임 유형: 스토어 / CXM (선택)
  - 고객명, 제품코드(SAP), 판매수량
  - 주소1(시/구), 상세주소, 우편번호, 전화번호
  - 국가코드 (KR 또는 해외)
  - 유통경로 (10: 국내 / 20: 해외)

[처리]
- 입력된 데이터를 구글 시트 "클레임_통합리스트"에 행 추가
- Slack 채널에 접수 알림 메시지 전송
- 클레임 유형이 "스토어"이고 주소 정보가 있으면 → 재출고 처리로 분기
- 클레임 유형이 "CXM"이면 → 바로 재출고 처리로 분기
- 재출고 처리:
  - SAP 업로드용 엑셀 파일을 자동 생성 (판매오더 Upload 형식)
  - 생성된 파일을 OneDrive "/SAP_Upload/" 폴더에 저장

[출력]
- 엑셀 파일: 한 건당 하나의 .xlsx 파일
  - 컬럼: 출하요청일, 유통경로, 제품코드, 판매수량,
          고객명, 주소, 우편번호, 전화번호, 국가코드

[전달]
- 파일 생성 완료 시 Slack 채널에 "파일 생성 완료" 알림
- 국가코드가 KR이면 → Slack에 "국내 출고 완료" 알림만 발송
- 국가코드가 KR이 아니면 → Global logistics 팀에게 자동 이메일 발송
  - 수신: global-logistics@company.com
  - 제목: "[해외출고요청] {고객명} - DO #{DO번호}"
  - 본문에 상품정보, 수령인 정보, DO 번호를 포함
```

## 프롬프트를 더 좋게 만드는 팁

### 1. 예외 상황을 알려주세요

```
[예외 처리]
- 첨부파일이 없는 이메일은 무시해줘
- 엑셀 파일이 비어있으면 "데이터 없음" 알림만 보내줘
- 구글 시트 연결이 실패하면 Slack에 에러 알림을 보내줘
```

→ AI도 "만약에..."를 미리 생각할 수 있습니다.

### 2. 실제 데이터 예시를 주세요

```
[데이터 예시]
엑셀 파일 안의 데이터는 이런 식이야:

| 주문번호 | 고객명 | 상품코드 | 수량 | 금액 |
|---------|--------|---------|------|------|
| ORD-001 | 김철수 | SKU-A01 | 3    | 45000|
| ORD-002 | 이영희 | SKU-B02 | 1    | 15000|
```

→ AI가 데이터 구조를 정확히 이해하고, 더 정확한 워크플로우를 만들 수 있습니다.

### 3. 현재 수작업 과정을 그대로 적으세요

```
[현재 수작업 과정]
1. 매일 아침 Outlook을 열어서 "주문현황" 메일을 찾는다
2. 첨부파일을 다운로드한다
3. 엑셀을 열어서 데이터를 복사한다
4. 구글 시트를 열어서 맨 아래 행에 붙여넣기 한다
5. 오늘 날짜를 A열에 적는다
6. Slack에 "업데이트했습니다"라고 메시지를 보낸다
```

→ AI가 각 단계를 n8n 노드로 1:1 변환할 수 있습니다.
   이게 가장 쉬우면서도 효과적인 방법입니다.

### 4. 단계적으로 요청하세요

한 번에 완벽한 프롬프트를 쓸 필요 없습니다.

```
1차: "이메일 오면 구글 시트에 기록하는 n8n 워크플로우 만들어줘" (기본 골격)
2차: "거기에 Slack 알림 추가해줘" (기능 추가)
3차: "첨부파일이 없을 때 에러 처리도 넣어줘" (예외 처리)
4차: "국내/해외 분기 처리도 추가해줘" (분기 로직)
```

→ 대화하듯이 점점 발전시키면 됩니다.

### 5. 결과물 형식을 명확하게 지정하세요

```
[결과물 형식]
- n8n에서 바로 import할 수 있는 JSON 형식으로 만들어줘
- 각 노드에 한글로 된 설명(description)을 넣어줘
- 테스트할 수 있도록 더미 데이터도 같이 만들어줘
```

## 한눈에 보는 비교표

| 구분 | 나쁜 프롬프트 | 좋은 프롬프트 |
|------|-------------|-------------|
| 길이 | 한 줄 | 20~40줄 |
| 상황 | 없음 | 팀, 업무, 현재 문제점 |
| 입력 | "이메일" | 어떤 이메일, 어떤 조건, 어떤 형식 |
| 처리 | 없음 | 필터, 계산, 분기 조건 |
| 출력 | "엑셀" | 파일명, 컬럼, 형식, 위치 |
| 전달 | 없음 | 누구에게, 어떤 채널, 어떤 메시지 |
| 예외 | 없음 | 에러 시 알림, 빈 데이터 처리 |
| 예시 | 없음 | 실제 데이터 샘플 |

## 최종 요약

```
좋은 프롬프트 = 신입사원 인수인계서

1. 어떤 업무인지 배경을 알려주고 (상황)
2. 데이터가 어디서 오는지 알려주고 (입력)
3. 어떻게 처리해야 하는지 알려주고 (처리)
4. 결과물이 뭔지 알려주고 (출력)
5. 누구에게 전달하는지 알려주세요 (전달)

+ 예외 상황과 데이터 예시를 주면 금상첨화!
```

**"내가 지금 하는 수작업을 그대로 적는 것"이 가장 좋은 프롬프트입니다.**
